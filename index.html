Blackberry.js

A zero-build framework for the artisanal web.

Getting Started

Blackberry.js lets you create reusable components using Alpine.js, with scoped styles and a composable component model, without a build step or additional dependencies. 

        	
<template blackberry tag="counter-component" props="initial">
	<script setup>
		$data.count = Number($props.initial ?? 0)
	 	$effect(() => console.log('count changed', $data.count))
	  	$cleanup(() => console.log("bye"))
	</script>

	<markup>
		<button @click="count++">count: <span x-text="count"></span></button>
	</markup>

	<style scoped>
		button { color: red; }
	</style>
</template>

<counter-component initial="10"></counter-component>

Key Features

  • Scoped Styling: Keep your CSS isolated to components
  • Reactive Data: Automatic updates when your data changes
  • Lifecycle Hooks: Setup and cleanup made easy
  • Props System: Pass data between components seamlessly

Component Structure

Each Blackberry component consists of three optional sections:

<style scoped>

Define component-specific styles that won't leak to other elements.

<script setup>

Initialize component data and define reactive behaviors using special variables:

  • $data: Component's reactive state
  • $props: Input properties
  • $effect: Run side effects when dependencies change
  • $cleanup: Clean up resources when component unmounts

<markup>

Define your component's HTML structure with reactive bindings:

  • @click: Event binding
  • x-text: Text content binding
  • x-model: Two-way data binding
